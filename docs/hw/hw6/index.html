    Homework 6

    Note

    This homework has two tasks: Task 1 and Task 2. You should start working on Task 1 now; Task 2 will be made available after Tue 10/12, once we have covered "deployment" and "continuous integration" concepts in class.

    caution
    • You are expected to work on this homework as a team in your (project) group.
    • Both Task 1 and Task 2 Due: Friday October 15th at 11pm EST (Baltimore time).
    • This assignment is worth 50 points.

    Learning Objectives#

    Objectives
    • Front-end styling, incorportating logic into front-end
    • Gaining (more) familiarity with HTML, CSS and Javascript

    Task 1#

    You will be working with CSS and JavaScript to improve the front-end of JBApp.

    Grab a copy of homework 6 starter code, under hw6_starter folder, from the JBApp repo on jhu-oose github organization. In class, we applied some basic styling to our html pages in the JBApp. We also warmed up with JavaScript to disallow empty usernames and also added delete buttons for each employer in the employers view. Your task is to:

    1. Add the add!: Currently, employers view shows a list of all the employers from the employers table where we can remove any of the listed employers using the delete button (i.e. button labled X) next to it. Let's improve this page such that we can do employer insertion also. Update employers page so that empty text fileds get shown at the very bottom after all the displayed rows where the user can type in an employer info to be inserted into the table. To the left of the text field, there must ba label "Add new: " and to the right of them there should be a little button similar to the delete button, but with the text of a checkmark character âœ….

    After a new employer is added, the page should refresh showing the newly added employer and the empty textfields and the add button at the bottom.

    tip

    You can use the html code &#10004; to produce a checkmark.

    Implementation Tip

    HINT: One way to implement this is to create the label, three textboxes, and a submit button as one HTML form at the bottom of employers.vm page after all employers are shown! Next, set the "onsubmit" attribute of the form tag to a Javascript function that you write in employers.js where you send a HTTP Post request with the form data to the HTTP Post employers endpoint. Once the HTTP Post is done then refresh the page (similar to what already is done in deleteEmpoloyer).

    1. Validate!: Under the src/main/resources/public/js, in the validate.js write new a function (or functions) to make sure that the provided inputs are valid when adding an employer:

      • An employer name is considered valid if it is not less than two characters, not longer than 150 characters and does not have any of the $, @, ^, %, ~ characters in it. If employer name is invalid, show a dialogbox with the message: "Employer name cannot be less than 2 characters, more than 150 characters, or include $, @, ^, %, ~!"
      • An employer sector cannot be less than two characters, more than 100 characters and may not contain any digits or any of the $, @, ^, %, ~ characters.
      • No input validation needed on the summary field!

    2. Finish in style!: Write your own css file to be applied to the html views of the JBApp. You can be creative here! Show us your artsy genes! :). It does not have to be a complex styling; write some basic styling for all the pages comparable to what we did in class. The css file we used in class is given as part of the starter code for your reference, but you should write your own css from scratch. At the very least, apply some stylings to the following components:

      • all textboxes in all the pages
      • all "submit" buttons in all the pages
      • all heading titles(h1 through h5 tags) in all the pages
      • all hyperlinks in all the pages
      • all labels in all the pages
      • for the delete buttons in the employers view, make the buttons circle-shaped and colored red; color the X text white! You can use any sizing you want for the circles/text, but do not make them overly small/big!
    caution

    You may not use any styling framework/library here. You need to write vanilla css (i.e., pure css).

    caution

    All stylings must be done in a separate css file name myjbapp.css. Do not do any styling in the html/vm files. Make sure to place the myjbapp.css under src\main\resources\public\css and load it into top.vm.
