    Homework 1

    caution
    • You are expected to work individually.
    • Due: Wednesday September 8th at 11pm EST (Baltimore time).
    • This assignment is worth 30 points.

    Learning Objectives#

    Objectives
    • Writing SRS document
    • Gradle, Java, OkHttp, HTTP, JSON, SIS API, Client-Server
    • CRUD vs. above & beyond CRUD
    tip

    It would make sense for you to clone your empty 'jhu-oose's private repo before attempting any of these tasks. Create a folder named hw1 in the repo with three subfolders named hw1-task1, hw1-task2, and hw1-task3 in it, and push it to your Github's repo. As you make progress and work through the tasks, you can then make periodic pushes to save your progress.

    Task 1#

    If you have already selected a project topic (or considering a few alternatives), make this about your topic; write the "requirement specification" accordingly (but do so individually). If you don't have a topic in mind, you can make it about a project idea you might already have in mind or you come up with. Your specification document must include all sections outlined in the SRS template. You need to do this in two different formats: PDF and Github Markdown. See Submission sections below for details.

    tip

    Open the SRS doc, then go to File -> Make a Copy where you will be able to edit the document. Once finished, go to File -> Download -> PDF document.

    tip

    Markdown is a very simple markup language for creating formatted text using an editor. You can learn about Github's flavor of Markdown here.

    tip

    For user stories (i.e. functional requirements), make sure you see User Stories section before you start working on them.

    Save Your SRS

    Save your SRS document in both PDF and Markdown formats. Name these files srs.pdf and srs.md and place them inside a folder named hw1-task1 within hw1.

    Task 2#

    Utilizing OkHttp and similar to what we did in sis-class-search app, write code to send http requests to SIS API's endpoints to find out the followings:

    1. All classes where the course title contains the words "software engineering".
    2. All fall 2021 classes that the course title contains the words "software engineering".
    3. All 3 credit, fall 2021 classes where the course title contains the words "software engineering".
    4. All 3 credit, fall 2021 classes where the course title contains the words "software engineering" and has open seats.
    caution

    For each of the above, you need to compose and dispatch/collect a separate http request/response.

    Print the results of each request to the standard output.

    Create a new Gradle Project

    For this task, create a fresh Gradle Java project named hw1-task2. This will create a folder named hw1-task2 where all your project files are inside it. Within hw1-task2 project, write your code in a class named Main. Aside from the public static void main function, feel free to write helper functions as needed in Main class; modular programming is always a good practice! hw1-task2 must be nested inside hw1.

    Task 3#

    Assume we have the following paragraph describing a cooking app named MyCookingApp:

    "The goal of this project is to create a web application where we can view, remove, post, update and search among cooking recipes. The app allows its user to view, modify, and share new cooking recipes with other users. Also, it allows for searching among all recipes using different filters e.g., title keywords, ingredients, recipe type, etc.

    Answer the followings:

    1. Is MyCookingApp a CRUD application or does it go beyond CRUD? why or why not?
    2. If MyCookingApp is a CRUD app, come up with at least two ideas how to make this app go above and beyond CRUD?
    3. Try and see if you can come up with a novel idea to make MyCookingApp stand out from existing solutions? In other words, try to make MyCookingApp different from the other popular cooking apps, such as Allrecipes or BigOven, that are already out there.
    tip

    Note that part 2 and 3 of this task are open-ended questions. You might want to play around a little with existing cooking apps. Keep in mind though you do not need to spend lots of time on this, just try to get a sense of what existing solutions accomplish and then put your own spin into it. The goal here is not to come up with the next million-dollar bussiness idea; rather just to practice brainstorming new ideas in the context of a real and tangible problem.

    Save Your Answers

    Save your answers to Task 3 questions in a file named task3.md and place it in the folder named hw1-task3 inside hw1 folder.
