    2. Add a job#

    Add a new link to the front page of JBApp titled "Add a job". When the link is clicked it should go to /addjobs view to show an HTML form that looks like the following:

    Requirements:

    • Title is a text box.
    • Employer is a dropdown list that lists all the employers' names that exist in the employers table.
      caution

      Note that this dropdown list is dynamically populated based on the records saved in the employers table.

    • Date posted and Deadline are dates!
    • Domain is a textbox.
    • Location is a textbox.
    • Full-time is a dropdown list that has two options: Yes corresponding to true and No corresponding to false. Default is Yes.
    • Salary/Hourly is a dropdown list with two options: Salary corresponding to true and Hourly corresponding to false. Default is Salary.
    • Payment is a number textbox (hint: use type "number") with the default value of $100000 (hint: use placeholder attribute)
    • Requirements is a textarea.

    When the button Add is clicked, the content of the form must be posted to the jobs API endpoint (i.e. via HTTP Post method) to be saved in the jobs table. This means you need to have a HTTP Post /jobs endpoint that accepts a post request to insert a job in the jobs table. This endpoint returns the inserted job as a JSON upon successful insertion, so you should see a JSON after Add is clicked (assuming things are implemented correctly!)

    tip

    Note that the employer name is selected from the dropdown list. This means the form only facilitates adding a job for an employer that already exists in the employers table. Though, when doing the insertion in the jobs table, you may need to first fetch the employer record from the employers table.

    tip

    When working with the dates and implementing HTTP Post jobs endpoint, you might find java.text.SimpleDateFormat handy to convert the dates to the desired format suitable for insertion into jobs table.

    Naming Requirement

    The view for this part must be named addjobs.vm. Also, make sure to name the new HTTP Post API endpoint jobs.

    3. Search jobs#

    Here you will implement a super simple searching functionality for JBApp. Clicking Search jobs link on the front page will take us to /search view that looks like the following:

    User can enter one or more keywords here to search among available jobs saved in the jobs table. Here is how the search needs to be done:

    • Search phrase is consisted of one or more keyword(s). If there are mutiple keywords, they are considered as part of one single phrase!
    • Any jobs whose title contains the search phrase (as a single phrase of one or more consecutive words) in its entirety, must be included.
    Examples
    • Example 1: if the search phrase is "software" all jobs with the title "Software Engineer" must be included!
    • Example 2: if the search phrase is "software engineer", a job with the title "Software Developer" is NOT included!
    • Any jobs whose domain contains the search phrase (as a single phrase of one or more consecutive words) in its entirety, must be included.
    Examples
    • Example 1: if the search phrase is "Health" all jobs with the domain "Health Care" must be included!
    • Example 2: if the search phrase is "software engineering", a job with the domain "Software Engineering/Development" must be included!
    • Any jobs whose company name matches exactly the search phrase (as a single phrase of one or more consecutive words) in its entirety, must be included.
    caution

    All the above search matchings must be done case-insensitively!

    The result of the search must be returned as a JSON array that includes all the matching jobs as described above!

    tip

    You would probably need to add two API endpoints for this part: a HTTP Get /search and HTTP Post /search. HTTP Post /search is the one that receives the keywords and does a search and returns the JSON array that contains all the matching jobs!

    Naming Requirement

    The view for this part must be named search.vm. Also, make sure to name the new HTTP Post and HTTP Get API endpoints /search.

    5. JUnit Tests#
